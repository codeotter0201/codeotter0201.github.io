個人簡介

經歷

我今年 31 歲，主要使用的語言是 Python ，程式開發經驗為 2 年。我在過去的學經歷主要都是業務相關，這些過程讓我學習到團隊合作和人與人之間溝通的重要性。

動機

而在這幾年的工作經驗中，難免會碰到數據或是軟體相關的處理與調整，我在這些偶然的機會當中，發現自己對於軟體工程十分有興趣，確定自己的想法後決定認真朝這條道路邁進。

專案作品

個人獨立開發專案，串接 Shioaji API，利用 Docker 分別容器化行情資料、策略運算、網頁渲染服務，容器化的架構讓整個專案具有可靠性、擴展性、有效率等等優勢，並運用 Github 進行版本控制，使專案維護的工作更為穩健。

Github

專案特色：

  • 即時行情Tick緩存於Redis，並將落地數據利用Pickle於本地持久化
  • 數據運算，搭配Numba技術，讓Python運算效能接近 C 的運算速度
  • Line notify觸發事件通知，掌握服務的即時狀態
  • 資料使用Plotly視覺化，提高數據報告的驗證效率
  • 後端，Fastapi與RESTful API 設計模式，做為各服務之間的溝通管道
  • 前端，使用Streamlitt呈現易用簡潔的網頁服務，並適用於多種設備
  • 部屬於Linode，每月10USD成本即可運作24小時的雲端監控服務

    我目前是全職準備轉職，可使用的時間比較充裕，但我打算盡量讓自己維持正常上下班的作息，例如週末時間與朋友家人相處，平日 09:00 ~ 18:00 鑽研專案這樣子的排程

    軟體工程的技術迭代很快，為了更有效率的學習，有一套適合自己的學習工作流，是我認為很值得做的項目之一。

    所以我打算好好記錄學習過程中遇到的困難，並且輸出一些有用的技術筆記。現階段是利用 Obsidian 蒐集資訊、整理，最後輸出成文章筆記。

    是否有想要加入的軟體公司？為什麼想加入該公司？

    我對於台灣知名軟體公司的認識不多，FAANG 對我來說又太過遙遠了，所以如果我有一個很想加入的軟體公司，我會希望：

  • 核心業務有護城河
  • 清晰的制度規範
  • 敢於嘗試，同時也知道該如何有效地嘗試
  • 大部分在公司工作的同事能找到成就感、歸屬感

    • 請描述一件讓你產生明顯負面情緒的事情，你如何處理該情緒？

    我曾在過去工作時，依照主管的指示執行任務，但最後任務失敗，大老闆怪罪下來，主管卻拿我當戰犯，而我在執行之前也盡可能跟主管確認各種方案，最後得到的結局是我被移除職位。

    第一時間意識到結果時當然是很難接受，只能盡量讓自己抽離，避免魯莽的做任何決策，待自身情緒真的穩定後再來抽絲剝繭。

    我覺得讓理性與感性分開平衡，是很重要的一步，但實際上並沒有這麼容易。

    不過很值得替開心的是，也因為有這經歷，讓我去思考遇到不會解的問題時，應該用甚麼態度面對。

    儘管我再次遇到類似的狀況，心情還是會受影響，但我可以確定我不會因此被擊倒。

    關於這份申請網頁，請分享一個你開發時的技術心得。

    起初我是完全不會寫前端(html&CSS)，我首先看了彭彭老師的基本html課程，但是看完之後還是有點不知道怎麼下手， 做中學是對我來說比較有感的方式，最後我決定直接找看看有沒有類似的教學可以參考，並且直接用教學提供的 Code 來做修改。 實作過程中，儘管是改已經完成的作品，但還是有些不知道如何調了更好，為何這樣寫的部分，以下是我有感覺但還需要學習的點。

  • CSS 的樣式設定不夠熟悉，也不太了解標籤之間的組合該如何調整
  • HTML 的基本骨架設定經驗不足，無法有效快速的創立一套好讀好改的程式碼
  • 閱讀理解程式碼的速度有待加強，基本常識不夠
